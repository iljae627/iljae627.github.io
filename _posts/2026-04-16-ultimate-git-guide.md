Git은 단순히 소스코드를 저장하는 도구가 아니라, 파일 시스템의 **스냅샷(Snapshot)**을 관리하고 팀 프로젝트의 **히스토리**를 제어하는 핵심 인프라입니다. 여러 가이드를 분석하여 가장 효율적이고 직관적인 Git 명령어 체계를 정리했습니다.

---

## 1. 커밋과 브랜치의 본질 (The Basics)

Git의 모든 작업은 **커밋**이라는 스냅샷을 찍고, **브랜치**라는 포인터로 그 흐름을 관리하는 것에서 시작합니다.

- **Commit**: 프로젝트의 특정 시점을 저장하는 단위. 이전 커밋(부모)을 가리키며 연결되어 히스토리를 형성합니다.
- **Branch**: 커밋 이력 위를 돌아다니는 가벼운 **참조(Reference)**일 뿐입니다. 생성과 삭제가 매우 자유롭습니다.
- **HEAD**: 현재 내가 작업 중인 위치를 가리키는 포인터입니다.



| 명령어 | 설명 |
| :--- | :--- |
| `git commit -am "msg"` | 추적 중인 파일의 스테이징과 커밋을 동시에 수행 |
| `git switch -c [name]` | 새 브랜치 생성과 동시에 이동 (최신 문법) |
| `git branch -f main HEAD~3` | main 브랜치를 현재 위치 기준 3단계 전으로 강제 이동 |

---

## 2. 히스토리 병합 전략: Merge vs Rebase

나누어진 작업 흐름을 하나로 합칠 때, 프로젝트의 성격에 맞는 전략 선택이 중요합니다.

- **Merge**: 두 브랜치의 공통 조상을 찾아 새로운 **병합 커밋**을 만듭니다. 작업 이력이 남아야 하는 공용 브랜치 병합에 유리합니다.
- **Rebase**: 커밋들을 대상 브랜치 위로 **재배치**하여 히스토리를 한 줄의 선형 구조로 만듭니다. 커밋 로그가 깔끔해지지만, 이미 공유된 커밋에 사용하면 충돌을 유발하므로 주의가 필요합니다.



---

## 3. 정교한 위치 제어와 취소 (Control & Undo)

### **상대 참조 (Relative Refs)**
해시값을 직접 입력하는 대신 `^`(한 단계 위)와 `~<num>`(지정한 수만큼 위)을 사용하여 트리 위를 빠르게 이동할 수 있습니다.

### **변경 되돌리기: Reset vs Revert**
- **Reset**: 브랜치 포인터를 과거로 직접 이동시킵니다. **로컬 작업**을 깔끔하게 정리할 때 사용합니다.
- **Revert**: 실수를 취소하는 **새로운 커밋**을 만듭니다. 이미 **원격에 공유된 작업**을 안전하게 되돌릴 때 유일하게 권장되는 방식입니다.

---

## 4. 고급 조작 테크닉 (Advanced)

### **Cherry-pick**
다른 브랜치에 있는 특정 커밋만 쏙 골라 현재 위치(`HEAD`) 아래에 복사본을 만듭니다. 전체를 합치지 않고 핵심 기능만 가져올 때 유용합니다.

### **Interactive Rebase (-i)**
커밋의 순서를 바꾸거나, 여러 커밋을 하나로 합치고(Squash), 불필요한 커밋을 삭제(Drop)하는 강력한 도구입니다. 원격 Push 전 히스토리를 정제하는 데 필수적입니다.

### **Stash**
작업 중인 내용을 임시 저장소에 잠시 치워둡니다. 브랜치를 급하게 전환해야 할 때 유용하며, `git stash pop`으로 언제든 복원할 수 있습니다.

---

## 5. 원격 저장소와 협업 (Remote & Synchronization)

원격 저장소(origin)는 팀원과의 공유 접점입니다.

- **Fetch**: 원격의 데이터를 다운로드하지만 로컬 파일은 건드리지 않습니다. (`origin/main` 갱신)
- **Pull**: `fetch` 후 곧바로 현재 브랜치와 병합합니다. (`--rebase` 옵션을 쓰면 히스토리가 더 깔끔해집니다.)
- **Push**: 내 로컬 커밋을 원격에 업로드합니다. 반드시 `pull`을 먼저 하여 원격의 최신 상태와 맞춘 뒤 수행해야 합니다.

---

## 6. 이정표 남기기: Tag & Describe

- **Tag**: 커밋이 늘어나도 절대 움직이지 않는 **닻(Anchor)**입니다. 주로 v1.0 같은 배포 버전에 사용합니다.
- **Describe**: 현재 위치가 가장 가까운 태그로부터 얼마나 떨어져 있는지 알려줍니다. 
  - *출력 형식: `<tag>-<numCommits>-g<hash>`*
