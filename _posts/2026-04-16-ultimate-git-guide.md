Git은 작업 기록을 커밋으로 남기고, 브랜치로 흐름을 나누고, 원격 저장소로 작업을 공유하는 도구입니다. 명령어가 많아 보여도 실제로는 **저장, 이동, 합치기, 되돌리기, 공유**라는 5가지 흐름으로 묶어서 이해하면 훨씬 쉽습니다. **Learn Git Branching** 실습을 통해 정리한 핵심 내용을 기록합니다.

---

## 1. 기본 작업: 저장 (Commit & Branch)
Git의 가장 기본 단위는 **커밋(Commit)**입니다. 커밋은 프로젝트의 특정 시점을 저장한 스냅샷이며, 각 커밋은 고유한 해시값을 가집니다.

- `git commit`: 현재 변경 내용을 새 커밋으로 저장.
- `git branch [이름]`: 새 브랜치 생성. 브랜치는 특정 커밋을 가리키는 포인터입니다.
- `git checkout [이름]`: 해당 브랜치로 이동.
- `git checkout -b [이름]`: 브랜치 생성과 이동을 동시에 수행.



---

## 2. 브랜치 합치기: 병합 (Merge & Rebase)
나누어 작업한 내용은 다시 합쳐야 합니다. 상황에 맞는 전략을 선택합니다.

- **Merge**: 현재 브랜치에 다른 브랜치의 작업을 합칩니다. 서로 다른 작업 흐름을 하나로 합칠 때 사용합니다.
- **Rebase**: 현재 브랜치의 커밋을 대상 브랜치 위로 다시 쌓습니다. 커밋 기록을 직선 형태로 정리할 때 유용합니다.

---

## 3. 커밋 이동: 제어 (HEAD & Relative Refs)
**HEAD**는 현재 내가 보고 있는 위치를 가리키는 포인터입니다.

- `HEAD^`: 부모 커밋으로 이동.
- `HEAD~3`: 현재 위치에서 3단계 이전 커밋으로 이동.
- `git branch -f main HEAD~3`: `main` 브랜치 포인터를 현재 기준 3단계 이전으로 강제로 이동시킵니다.



---

## 4. 변경 되돌리기: 취소 (Reset & Revert)
상황에 따라 되돌리는 방식을 다르게 선택해야 안전합니다.

- **Reset**: 브랜치를 이전 커밋으로 되돌립니다. 로컬에서 아직 공유하지 않은 작업을 정리할 때 사용합니다.
- **Revert**: 변경을 취소하는 **새로운 커밋**을 만듭니다. 이미 원격에 올렸거나 공유한 커밋을 되돌릴 때 안전합니다.
  - *Tip: `--soft`는 스테이징 유지, `--hard`는 모든 변경사항 삭제.*

---

## 5. 원격 저장소: 공유 (Remote & Collaboration)
서버(GitHub 등)에 있는 저장소와 데이터를 주고받는 과정입니다.

- `git clone [url]`: 원격 저장소를 로컬로 복제.
- `git fetch`: 원격의 새 커밋을 가져오되, 로컬 브랜치나 파일은 직접 바꾸지 않습니다.
- `git pull`: 원격 변경을 가져오고 합칩니다. (`fetch` + `merge`)
- `git push`: 로컬 커밋을 원격에 업로드하여 작업을 공유합니다.

---

## 6. 특정 커밋 옮기기 및 태그 (Advanced)

- **Cherry-pick**: 필요한 특정 커밋만 골라서 현재 위치에 복사합니다.
- **Interactive Rebase (`-i`)**: 최근 커밋들을 UI를 통해 대화형으로 정리(순서 변경, 삭제 등)합니다.
- **Tag**: 특정 커밋에 버전 이름(v1.0 등)을 영구적으로 표시합니다. 커밋이 추가되어도 절대 움직이지 않는 '닻' 역할을 합니다.
